## 🔧 技术配置

### fstab 自动挂载配置
```bash
//192.168.180.162/SSD_960GB /mnt/nas_share cifs credentials=/root/.smb_credentials,vers=3.0,uid=1000,gid=1000,file_mode=0755,dir_mode=0755,nofail,_netdev 0 0
//192.168.180.162/外接存储-HGST /mnt/nas_hgst cifs credentials=/root/.smb_credentials,vers=3.0,uid=1000,gid=1000,file_mode=0755,dir_mode=0755,nofail,_netdev 0 0
//192.168.180.162/外接存储-TOSHIBA /mnt/nas_toshiba cifs credentials=/root/.smb_credentials,vers=3.0,uid=1000,gid=1000,file_mode=0755,dir_mode=0755,nofail,_netdev 0 0
//192.168.180.162/外接存储-WDC /mnt/nas_wdc cifs credentials=/root/.smb_credentials,vers=3.0,uid=1000,gid=1000,file_mode=0755,dir_mode=0755,nofail,_netdev 0 0
```

### 关键发现
1. **共享类型决定可访问性**: 只有 `Disk` 类型共享可访问
2. **外接存储显示为硬盘型号**: 说明是物理设备识别，不是文件共享
3. **需要在飞牛 NAS 上启用 SMB 共享**: 这是唯一缺失的步骤

## 💡 经验教训

1. **NAS 共享需要双重配置**: 硬盘识别 ≠ 文件共享启用
2. **飞牛 fnOS 的特殊性**: 需要 Web 界面操作启用共享
3. **`nofail` 选项的重要性**: 确保系统启动稳定性
4. **用户友好链接的价值**: `~/nas*` 链接极大简化访问
